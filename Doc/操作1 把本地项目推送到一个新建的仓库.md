# 从零开始用Git管理项目的完整指南

这是一份**无遗漏、可直接照着操作**的Git项目初始化全流程，适用于任何语言的新项目或已有项目，覆盖从本地仓库创建到远程同步的所有步骤。

**注意：**极高效地学用Git：10分钟了解Git的核心概念，知道本地仓库、本地缓存、远程仓库、pull、push这些，就可以指挥Trae去实现了，比如用下面的Prompt：
我在Gitcode中创建了一个TP\_IotGateway空仓库，把本项目上传到Gitcode的TP\_IotGateway空仓库中，下面是Gitcode的TP\_IotGateway仓库的网址：
https://gitcode.com/u012915636/TP_IotGateway.git

## 一、准备工作（仅首次使用Git需要）

### 1. 安装Git

- **Windows**：从 [Git官网](https://git-scm.com/download/win) 下载安装，全部默认选项即可
- **macOS**：终端运行 `xcode-select --install` 自动安装
- **Linux**：`sudo apt install git` (Ubuntu/Debian) 或 `sudo dnf install git` (CentOS/Fedora)

验证安装是否成功：

```bash
git --version
# 输出类似 git version 2.45.1 即表示成功
```

### 2. 配置全局身份（必做）

这是你在Git提交记录中的唯一标识，所有提交都会带上这个信息：

```bash
# 配置用户名（建议用你的GitHub/Gitee用户名）
git config --global user.name "你的用户名"

# 配置邮箱（必须和你远程仓库的注册邮箱一致）
git config --global user.email "你的邮箱@example.com"

# 验证配置
git config --list
```

## 二、第一步：创建远程仓库（推荐先做）

**强烈建议先在代码托管平台创建空的远程仓库，再关联本地项目**，这样可以避免后续的分支冲突问题。

### 在GitHub/Gitee/GitCode创建远程仓库

1. 登录平台，点击右上角「+」→「New repository」
2. 填写仓库名称（建议和本地项目文件夹同名）
3. 填写项目描述（可选）
4. ⚠️ **非常重要：不要勾选以下任何选项**
   - ❌ Initialize this repository with a README
   - ❌ Add .gitignore
   - ❌ Choose a license
5. 点击「Create repository」完成创建
6. 复制生成的远程仓库地址（推荐用HTTPS格式，新手更简单）

## 三、第二步：将本地项目纳入Git管理

### 情况A：已有现成的项目（最常见）

1. 打开终端，进入你的项目根目录：
   ```bash
   cd /path/to/your/project  # Windows: cd C:\path\to\your\project
   ```
2. 初始化本地Git仓库：
   ```bash
   git init
   ```
   > 执行后会在项目根目录生成一个隐藏的 `.git` 文件夹，这是Git的本地数据库，**绝对不要手动修改或删除它**
3. 创建 `.gitignore` 文件（必做！）
   这是Git管理中最重要的文件之一，用于指定哪些文件不需要被Git跟踪：
   ```bash
   # 创建.gitignore文件
   touch .gitignore
   ```
   编辑该文件，添加你需要忽略的规则。**推荐直接从** **[github/gitignore](https://github.com/github/gitignore)** **复制对应语言的标准模板**。

   通用基础模板示例：
   ```gitignore
   # 操作系统文件
   .DS_Store
   Thumbs.db

   # IDE和编辑器配置
   .idea/
   .vscode/
   *.swp
   *.swo

   # 编译生成的文件
   build/
   dist/
   *.class
   *.pyc

   # 依赖包
   node_modules/
   venv/
   __pycache__/

   # 敏感信息
   .env
   *.key
   *.pem
   ```
4. 将所有文件添加到暂存区：
   ```bash
   git add .
   # 或更安全的 git add -A（会包含删除的文件）
   ```
5. 提交到本地仓库：
   ```bash
   git commit -m "第一次提交：初始化项目"
   ```
   > 提交信息要简洁明了，说明这次提交做了什么
6. 关联远程仓库：
   ```bash
   # 将刚才复制的远程仓库地址粘贴到这里
   git remote add origin https://github.com/你的用户名/你的仓库名.git
   ```
7. 推送到远程仓库：
   ```bash
   # 第一次推送需要加 -u 参数建立分支跟踪关系
   git push -u origin main
   ```
   > 如果你用的是旧版Git，默认分支名可能是 `master`，请替换为 `git push -u origin master`

### 情况B：从零开始创建新项目

如果项目还没开始写，流程更简单：

```bash
# 克隆你刚才创建的空远程仓库
git clone https://github.com/你的用户名/你的仓库名.git

# 进入项目目录
cd 你的仓库名

# 开始写代码...

# 第一次提交
git add .
git commit -m "初始化项目"
git push origin main
```

## 四、验证是否成功

1. 刷新你的远程仓库页面，应该能看到所有本地文件都已经上传
2. 在本地运行以下命令，确认远程关联正常：
   ```bash
   git remote -v
   # 应该输出类似：
   # origin  https://github.com/你的用户名/你的仓库名.git (fetch)
   # origin  https://github.com/你的用户名/你的仓库名.git (push)
   ```

## 五、后续日常工作流（牢记这个循环）

以后每次修改代码后，都遵循这个标准流程：

```bash
# 1. 查看修改了哪些文件（每次提交前必做！）
git status

# 2. 添加修改到暂存区
git add 文件名  # 添加单个文件
git add .       # 添加所有修改

# 3. 提交到本地仓库
git commit -m "修复了用户登录的bug"

# 4. 推送到远程仓库
git push origin main
```

## 六、绝对不能做的事（新手最容易踩的坑）

1. ❌ **不要提交敏感信息**：密码、密钥、API密钥、数据库配置等绝对不能提交到Git仓库
2. ❌ **不要提交大文件**：超过100MB的文件Git不推荐提交，应该用Git LFS或其他方式存储
3. ❌ **不要提交编译生成的文件**：node\_modules、build、dist等文件夹都应该加入.gitignore
4. ❌ **不要手动修改.git文件夹**：里面的文件是Git自动管理的，手动修改会导致仓库损坏
5. ❌ **不要强制推送`git push -f`**：除非你非常清楚自己在做什么，否则会覆盖别人的代码
6. ❌ **不要在main分支上直接开发**：养成创建功能分支的习惯

## 七、常见问题与解决方案

### 问题1：远程仓库已经初始化了（有README、.gitignore）

如果你不小心勾选了初始化选项，推送时会报错：

```bash
error: failed to push some refs to 'https://github.com/xxx/xxx.git'
hint: Updates were rejected because the remote contains work that you do
hint: not have locally.
```

解决方案：

```bash
# 先拉取远程的更新并合并
git pull origin main --allow-unrelated-histories

# 然后再推送
git push -u origin main
```

### 问题2：提示 `fatal: remote origin already exists`

说明你已经关联过远程仓库了，先删除旧的再添加新的：

```bash
git remote remove origin
git remote add origin 新的远程仓库地址
```

### 问题3：不小心提交了敏感文件怎么办

参考我之前的回答：[更改.gitignore后如何同步到远程仓库](https://www.doubao.com/chat/7341514231459076)，需要使用 `git filter-branch` 或 BFG Repo-Cleaner 彻底清理历史记录。

## 八、最佳实践

1. **小步提交**：每次提交只做一件事，提交频率建议每1-2小时一次
2. **规范提交信息**：遵循 [约定式提交](https://www.conventionalcommits.org/zh-hans/v1.0.0/) 规范
   ```
   feat: 新增用户注册功能
   fix: 修复密码重置失败的bug
   docs: 更新README.md
   style: 格式化代码
   refactor: 重构用户模块
   ```
3. **使用分支管理**：开发新功能时创建 `feature/xxx` 分支，修复bug时创建 `bugfix/xxx` 分支
4. **定期拉取更新**：每天开始工作前先执行 `git pull origin main`，避免冲突
5. **维护更新日志**：在项目根目录创建 `CHANGELOG.md`，记录每个版本的变更
